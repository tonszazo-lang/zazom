# نظام الكاشير الذكي المتكامل

## 📋 نظرة عامة

تم تطوير نظام كاشير ذكي متكامل يربط جميع الأجزاء ببعضها بحيث تسمع التغييرات في كل مكان مع حفظ تلقائي للبيانات.

---

## 🔗 الترابط الكامل (Full Integration)

### 1. نظام الإشعارات الفوري (Real-time Notifications)

#### المكونات:
- **NotificationContext**: إدارة الإشعارات المركزية
- **NotificationCenter**: عرض الإشعارات العائمة
- **useNotification Hook**: الوصول إلى الإشعارات في أي مكان

#### الميزات:
```typescript
// استخدام الإشعارات في أي مكان
const { addNotification } = useNotification()

// أنواع الإشعارات
addNotification('تم إضافة المنتج', 'success')      // أخضر
addNotification('خطأ في العملية', 'error')         // أحمر
addNotification('حذر: كمية قليلة', 'warning')      // أصفر
addNotification('معلومة', 'info')                  // أزرق

// إشعارات مخصصة الوقت
addNotification('تم الحفظ', 'success', 2000)       // 2 ثانية فقط
addNotification('حذر مهم', 'warning', 0)            // بدون إزالة تلقائية
```

#### الإشعارات التلقائية:
- ✓ عند إضافة منتج للسلة
- ✓ عند إنشاء فاتورة
- ✓ عند تحديث المخزون
- ✓ عند إضافة مشتريات
- ✓ عند تسجيل إنتاج/استهلاك

### 2. نظام الحفظ التلقائي (Auto-Save System)

#### المكان المستودع:
```
Browser LocalStorage
├── pos_products        (المنتجات)
├── pos_cart           (السلة الحالية)
├── pos_invoices       (الفواتير)
├── pos_purchases      (المشتريات)
├── pos_productions    (الإنتاج)
├── pos_consumptions   (الاستهلاك)
└── pos_reports        (التقارير اليومية)
```

#### آلية الحفظ:
```typescript
// حفظ تلقائي عند أي تغيير
useEffect(() => {
  localStorage.setItem('pos_products', JSON.stringify(products))
}, [products])

// التحميل عند الدخول
useEffect(() => {
  const saved = localStorage.getItem('pos_products')
  if (saved) setProducts(JSON.parse(saved))
}, [])
```

#### المميزات:
- ✓ حفظ فوري بدون تأخير
- ✓ آمن تماماً (بيانات محلية)
- ✓ عدم فقدان البيانات
- ✓ سريع جداً

### 3. التحديثات الحية (Live Updates)

#### آلية العمل:

**عند بيع منتج:**
```
نقطة البيع (POS)
    ↓
updateProduct() [تقليل المخزون]
    ↓
localStorage يتحدث
    ↓
جميع الصفحات تسمع التغيير فوراً
    ↓
إشعار: "تم بيع 5 ماء معدني"
```

**عند شراء منتج:**
```
المشتريات (Purchases)
    ↓
addPurchase() [زيادة المخزون]
    ↓
localStorage يتحدث
    ↓
لوحة التحكم والمخزون يحدثان
    ↓
إشعار: "تم استيراد 50 منتج"
```

#### مثال عملي:
```typescript
// صفحة المشتريات
const handleAddPurchase = () => {
  addPurchase(purchaseData)  // يزيد المخزون
  addNotification('تم إضافة المشتريات', 'success')
  
  // جميع الصفحات تسمع التحديث:
  // - لوحة التحكم: المخزون يتحدث
  // - الحاسبة: تظهر المنتجات الجديدة
  // - POS: الكمية تتحدث
}
```

### 4. الحاسبة الذكية (Smart Calculator)

#### الميزات:
- ✓ حاسبة كاملة (0-9، نقطة عشرية)
- ✓ اختيار المنتج من قائمة
- ✓ إدخال الكمية مباشرة
- ✓ إضافة للسلة فوراً
- ✓ تحديثات فورية في السلة

#### الاستخدام:
```
1. اختر منتج من القائمة
2. ادخل الكمية بالحاسبة (مثل: 5)
3. اضغط "إضافة"
4. يتم إضافة المنتج للسلة فوراً
5. إشعار يظهر يؤكد الإضافة
```

#### التكامل مع النظام:
- ✓ تحقق من توفر الكمية
- ✓ تحديث تلقائي لـ POS
- ✓ حفظ فوري
- ✓ إشعارات ذكية

## 🎯 سيناريوهات الاستخدام

### السيناريو 1: بيع منتج عادي

```
الخطوة 1: POS → البحث والاختيار → أضف كمية
         ↓
         updateProduct() يقلل المخزون
         ↓
الخطوة 2: الإشعار يظهر: "تم إضافة المنتج"
         ↓
الخطوة 3: لوحة التحكم تحدث
         لوحة التحكم: المخزون قل من 50 → 49
         لوحة التحكم: مبيعات اليوم زادت
         ↓
الخطوة 4: عند الدفع، فاتورة جديدة تُنشأ
         ↓
الخطوة 5: التقرير اليومي يتحدث
```

### السيناريو 2: شراء من مورد

```
الخطوة 1: Purchases → أدخل المورد والمنتجات
         ↓
         addPurchase() يزيد المخزون
         ↓
الخطوة 2: الإشعار: "تم استيراد 100 منتج"
         ↓
الخطوة 3: جميع الصفحات تحدث:
         • Inventory: كمية زادت
         • Dashboard: إجمالي المخزون زاد
         • POS: المنتج متاح للبيع
         • Calculator: يمكن اختيار المنتج
         ↓
الخطوة 4: تقارير تحدث
```

### السيناريو 3: تسجيل خسارة (Consumption)

```
الخطوة 1: Consumption → اختر منتج وسبب الخسارة
         ↓
         addConsumption() يقلل المخزون
         ↓
الخطوة 2: الإشعار: "تم تسجيل 5 وحدات تالفة"
         ↓
الخطوة 3: المخزون ينخفض مباشرة
         ↓
الخطوة 4: تنبيه إذا انخفض دون حد معين
```

## 🔄 دورة التحديث الكاملة

```
1. المستخدم يؤدي فعل (click)
   ↓
2. دالة في المكون (handleAddToCart)
   ↓
3. دالة في StoreContext (addToCart)
   ↓
4. تحديث State (setCart)
   ↓
5. useEffect يحفظ في localStorage
   ↓
6. جميع المكونات المشتركة تسمع التغيير
   ↓
7. إعادة تصيير (Re-render) تلقائية
   ↓
8. إشعار يظهر (addNotification)
   ↓
9. لوحة التحكم والأجزاء الأخرى تتحدث
```
